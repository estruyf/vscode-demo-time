Demo API

Control your demos with simple HTTP requests

Getting Started

Welcome to the Demo API! This simple REST API allows you to control and trigger demos programmatically. All endpoints support GET requests and return JSON responses.

!

Tip

Add ?bringToFront=true to any endpoint to automatically bring the application window to the front.

API Endpoints

GET

List Demos

Returns all demo files, the next demo, and the currently executing demo file

Endpoint
/api/demos

  "currentDemoFile": "/Users/user/project/.demo/pitch.yaml"

Description
  • nextDemo: The next demo to be executed (title only), if available.
  • demos: Array of demo files with hierarchical structure. Each demo file contains children array with individual demo steps.
  • currentDemoFile: String path to the currently executing demo file.
  • children: Array of demo steps within each file, containing stepIndex, labels, execution status, and contextValue.
Usage Examples
{{API_URL}}/api/demos
GET

Trigger Next Demo

Triggers the next demo in the sequence

Endpoint
/api/next
Parameters
bringToFront boolean optional

Brings the application to the front

Example: true
Usage Examples
{{API_URL}}/api/next
{{API_URL}}/api/next?bringToFront=true
GET

Run Demo by ID

Runs a specific demo by its unique identifier

Endpoint
/api/runById
Parameters
id string required

The unique identifier of the demo to run

Example: demoId
bringToFront boolean optional

Brings the application to the front

Example: true
Usage Examples
{{API_URL}}/api/runById?id=demoId
{{API_URL}}/api/runById?id=demoId&bringToFront=true
